# GPT-Image-2 中文提示词使用指南

本文档面向中文用户，帮助你快速理解并使用本仓库中的 GPT-Image-2 图像生成提示词案例。它不是替代原有 README 的项目主页，而是补充一个更偏使用方法、模板拆解和中文写法的实用指南。

## 1. 项目内容概述

`awesome-gpt-image-2-prompts` 是一个 GPT-Image-2 提示词案例库，主要收集可复用的图像生成提示词、输出示例和任务场景参考。

从现有 README 和案例分区可以看到，仓库覆盖的图像类型包括但不限于：

- 人像与摄影类案例
- 海报与插画类案例
- 角色设计与角色卡案例
- UI、网页、应用界面和社交媒体截图案例
- 模型对比与社区实验案例
- 产品图、信息图、封面图、宣传图等扩展场景

对于中文用户来说，本仓库适合作为“提示词写法参考库”：你可以先找到接近自己需求的案例，再替换主体、场景、风格、比例、构图和限制条件，生成更贴合自己目标的图片。

## 3. GPT-Image-2 提示词的基本结构

一个高质量图像提示词通常不是一句简单描述，而是由多个信息层组成。你可以按下面结构组织：

1. 主体对象：画面中最重要的人、物、角色、产品、界面或场景。
2. 场景环境：主体所处的位置、时代、空间、天气、室内外环境。
3. 风格方向：写实摄影、电影海报、原创日系动画电影感、手绘背景、商业广告、现代 UI 等。
4. 光影氛围：自然光、逆光、柔光、硬光、霓虹、黄昏、清晨、体积光、通透光影等。
5. 构图方式：居中构图、三分法、低角度、俯视、对称构图、留白、近景、半身、全身等。
6. 镜头语言：35mm、50mm、85mm、人像镜头、广角、长焦、浅景深、手机抓拍、电影剧照感等。
7. 细节要求：皮肤质感、服装材质、背景纹理、文字区域、图标、道具、环境小细节。
8. 色彩要求：低饱和、高对比、暖色调、冷色调、蓝白清透、复古胶片、商业广告配色等。
9. 画面比例：如 `1:1`、`4:5`、`3:4`、`16:9`、`9:16`、横版或竖版。
10. 负面限制或避免项：不要变形、不要多手指、不要改变身份、不要错误文字、不要过度杂乱、不要套用具体受限艺术家名称等。

推荐写法是：先明确主体和目标，再补充风格、构图和限制。不要把所有想法都堆在一起，否则画面容易失去重点。

## 4. 人像照片渲染提示词模板

适合用户上传一张人物照片后，希望在保留人物身份特征的基础上进行风格化渲染、写真优化或视觉升级。

完整可复制模板：

```text
请基于我上传的人物照片生成一张高质量风格化人像图。

核心要求：
- 保留原人物身份特征，不改变人物是谁。
- 保留面部五官比例、脸型结构、眼睛形状、鼻子、嘴唇、眉毛等主要特征。
- 保留原有发型、发色、发量走向、头部姿态和身体姿态。
- 保留主要服装特征，包括衣服类型、颜色、领口、袖型和整体穿搭气质。
- 保留原表情的核心情绪，不改变年龄、性别和人物气质。

画面优化：
- 将背景优化为更干净、更有层次的环境，可根据照片氛围补充自然光、室内柔光或电影感背景。
- 提升画面质感，增强皮肤真实细节、服装材质、发丝层次和光影过渡。
- 使用自然、真实、克制的美化方式，不要过度磨皮，不要塑料质感。
- 构图保持人物为视觉中心，画面清晰、稳定、自然。

风格方向：
- 写实摄影质感，柔和电影级光影，浅景深，细腻但不过度夸张。
- 色彩自然协调，背景与人物气质统一。

避免项：
- 不要改变人物身份。
- 不要改变年龄、性别、脸型和五官比例。
- 不要生成夸张变形、卡通化过度、脸部崩坏、手部变形或肢体异常。
- 不要添加不相关人物，不要添加多余文字、水印或 logo。

画面比例：请生成【填写比例，例如 4:5 / 1:1 / 9:16】。
```

## 5. 原创日系动画电影感提示词模板

适合生成原创日系动画电影背景、清澈蓝天、细腻云层、通透光影、夏日海景、青春感和梦幻感的画面。

重要限制：不要使用具体在世艺术家、导演或工作室名称作为风格指令。请使用泛化描述，例如“原创日系动画电影感”“手绘动画背景”“高透明度光影”“夏日青春氛围”“细腻云层”“电影感构图”“唯美自然风景”。

完整可复制模板：

```text

画面主体：
- 【填写主体，例如：海边车站、临海坡道、白色灯塔、海岸公路、夏日教室窗边、骑自行车的少年少女】。

场景环境：
- 清澈蓝天，占据画面上方大面积空间。
- 细腻层次丰富的白色云层，云边缘被阳光照亮。
- 远处是通透的夏日海景，海面有细碎高光和柔和波纹。
- 画面中可以加入电线杆、路牌、栏杆、树影、白色建筑或远处小岛等生活化细节。

风格方向：
- 原创日系动画电影感。
- 手绘动画背景质感。
- 细腻自然风景，清新、安静、青春、梦幻。
- 画面具有电影感构图和叙事感，但不要模仿任何具体艺术家、导演或工作室。

光影与色彩：
- 高透明度光影，夏日午后阳光，空气清澈。

- 电影感横版或竖版构图，主体位于画面视觉焦点。

细节要求：
- 云层细腻，海面通透，树叶和建筑边缘有手绘背景的精致感。
- 整体画面唯美自然，不要过度奇幻，不要杂乱元素。

避免项：
- 不要使用具体在世艺术家、导演或工作室名称。
- 不要出现水印、错误文字、低清晰度、过度锐化、人物脸部崩坏或透视错误。

画面比例：请使用【填写比例，例如 16:9 / 9:16 / 4:5】。
```

## 10. 中文用户使用教程

建议按下面步骤使用本仓库提示词：

1. 选择图片类型：先明确你要做人像、海报、UI、角色卡、社交媒体封面，还是其他类型。
2. 找到对应案例：在 `README.md` 或 `README_zh-CN.md` 中查找相近案例，例如人像案例、海报案例、UI 案例或角色设计案例。
3. 替换关键信息：把原案例中的主体、场景、风格、比例、背景、色彩和镜头描述替换成自己的需求。
4. 上传参考图时说明保留哪些特征：如果使用人物、产品或已有界面参考图，要明确写出需要保留的身份特征、结构、颜色、布局或品牌元素。
5. 生成后继续迭代：根据结果补充问题，例如“人物不像”“背景太乱”“文字区域不够”“比例不对”“风格太重”“脸部变形”等，再进行二次修改。
6. 避免直接套用具体艺术家或在世创作者风格：可以使用泛化风格描述，例如“原创日系动画电影感”“复古胶片摄影”“商业海报设计感”“手绘动画背景”“现代极简 UI”。

实用建议：每次只重点优化 1 到 3 个问题，不要一次性追加太多新要求，否则模型可能顾此失彼。

## 11. 常见错误和优化建议

| 常见问题 | 表现 | 改进建议 |
| --- | --- | --- |
| 提示词太短 | 只写“生成一张好看的海报”或“帮我美化照片” | 补充主体、场景、风格、构图、光影、比例和避免项。 |
| 风格描述过于模糊 | “高级感”“好看”“大片感”没有具体方向 | 改成“低饱和商业摄影、柔和侧光、干净背景、主视觉居中、4:5 竖版”。 |
| 主体和背景冲突 | 人物、背景、道具互相抢画面 | 明确主视觉优先级，写清背景只做氛围，不要喧宾夺主。 |
| 没有说明保留人物特征 | 上传照片后生成结果不像本人 | 明确要求保留身份、五官比例、脸型、发型、姿态、服装和表情核心特征。 |
| 没有说明画面比例 | 输出尺寸不适合用途 | 根据用途指定比例：头像 `1:1`，小红书常用 `3:4` 或 `4:5`，短视频封面 `9:16`，网页展示 `16:9`。 |
| 使用了具体受限艺术家名称 | 风格请求不合适，且不利于原创表达 | 改用泛化描述，如“原创手绘动画背景”“电影感构图”“高透明度光影”“复古科幻海报”。 |
| 要求太多导致画面混乱 | 同时要人物、产品、城市、文字、特效、复杂背景 | 拆成主视觉、背景、文字区三层，每层只保留必要元素。 |
| 一次性合成多张图 | 想同时生成九宫格、海报、头像、详情页，结果不稳定 | 先生成单张核心图，再分别生成封面、详情页、角色卡或多图版本。 |
| 文字区域没有预留 | 后期加标题时遮挡主体 | 在提示词中明确“顶部留白用于标题”“右下角预留副标题区域”。 |
| 负面限制缺失 | 出现水印、乱码、多余人物、手部错误 | 加入避免项：不要水印、不要无关文字、不要多余人物、不要手指异常、不要低清晰度。 |
